## 📌 1. 백엔드(Back-end) 주요 보정 내역

백엔드 소스코드(`C:\inspire\miniproject_backend\demo`) 분석 중 컴파일 에러, 기능 미동작, 보안 구동 실패를 유발하는 치명적인 설계 결함들을 보정하였습니다.

### 1.1. AI 에이전트 프롬프트 포맷팅 누락 해결
* **기존 문제**: `TravelPlanAIAgent.java` 프롬프트 본문 내부의 매개변수 치환자 앞에 `%` 기호가 누락되어 `"s"`라는 일반 텍스트로 지정되어 있었습니다. 이 때문에 여행 시작일, 종료일, 필수 목적지들이 AI에게 전달되지 않고 오직 지역명만 들어가는 핵심 로직 버그가 발생했습니다.
* **보정 조치**: `"%s"`로 포맷 규격을 올바르게 수정하고, 목적지 객체 리스트를 쉼표(`,`) 구분 문자열로 안전하게 추출해 AI에게 완벽한 계획 수립 정보를 전달하도록 개선했습니다.

### 1.2. JWT 인증 필터 이중 실행 결함 제거
* **기존 문제**: `JwtAuthenticationFilter` 클래스 상단에 `@Component`가 명시되어 있으면서 `SecurityConfig`에 수동으로 `.addFilterBefore()` 등록이 되어 있었습니다. 이로 인해 모든 보안 접근 요청 시 필터가 중복(2번) 실행되어 DB/Redis 조회 오버헤드가 누적되었습니다.
* **보정 조치**: 필터 상단의 `@Component` 선언을 과감히 제거하여 글로벌 서블릿 컨테이너 자동 등록을 차단하고, `SecurityConfig` 필터 체인 내부에서 수동으로 완벽히 통제되도록 단일화했습니다.

### 1.3. CORS 전역 정책 부재 해결 (프론트엔드 연동 블로커)
* **기존 문제**: 스프링 시큐리티에서 `http.cors(Customizer.withDefaults())`를 명시했으나, 실제 CORS를 제어하는 `CorsConfigurationSource` 스프링 빈이 정의되어 있지 않았습니다. 로컬에서 프론트엔드(Vue/React 포트 5173, 3000 등)와 연동하면 CORS 거부 오류가 발생하여 통신 자체가 불가능했습니다.
* **보정 조치**: 와일드카드 및 자격 증명 전송(`allowCredentials`)을 안전하게 지원하는 `setAllowedOriginPatterns("*")` 전역 CORS 설정을 등록하고, 웹 클라이언트가 JWT 토큰을 획득할 수 있도록 `Authorization` 및 `Refresh-Token` 헤더를 외부 노출헤더(`ExposedHeaders`)로 지정했습니다.

### 1.4. 여행 계획 수정(`Update`) 로직의 날짜/시간 하드코딩 및 영속성 낭비 제거
* **기존 문제**: 수정 기능 수행 시, 상세 경로들의 날짜가 무조건 여행 전체 시작일로만 통일되어 들어가고 시각 또한 서버의 현재 시각(`LocalTime.now()`)으로 강제 덮어쓰기되는 데이터 훼손 현상이 있었습니다. 또한 JPA의 Cascade 영속성 전이 조건이 충족되었음에도 불필요한 `deleteAll()`, `saveAll()`이 중복 실행되었습니다.
* **보정 조치**: 프론트엔드가 수정 요청한 고유 일자 및 시간 데이터를 그대로 유지하도록 매핑 로직을 정상화하고, 부모 리스트 `clear()` 후 `add()` 하는 최적의 Cascade 구조로 영속성 로직을 단일화했습니다.

## 📌 2. 프론트엔드(Front-end) 주요 보정 내역

조원의 변경 전파 과정에서 발생한 디렉토리 명칭 파편화, 컴파일 에러, UI 연동 오작동을 전수 교정했습니다.

### 2.1. 디렉토리 명칭 단일화 및 이중 컨텍스트 충돌 정리
* **기존 문제**: 기존에 존재하던 `src/contexts/` 폴더 대신 단수형 이름인 `src/context/` 폴더를 새로 만들고 그 안에 새 `AccessTokenContext`를 정의하면서 컴포넌트 간 임포트 경로가 어긋나 컴파일 에러가 도처에 발생하고 있었습니다.
* **보정 조치**: 기존의 중복된 복수형 폴더(`contexts`)를 완전히 소거하여 구조를 깔끔하게 통합하고, 모든 컴포넌트(`TopAppBar` 포함)가 단수형 `context/`에 정의된 훅(`useAccessToken`)을 사용하도록 import 문을 통일했습니다.

### 2.2. 디스크에서 삭제되었던 필수 파일들 완전 복구 (Day.js 최적화)
* **기존 문제**: 조원의 로컬 작업 영역과 깃 관리 불일치로 인해 `api/axios.js`, `theme.js` 및 메인 페이지에서 필수 렌더링을 타는 `KoreanDatePicker.jsx`, `DestinationCard.jsx`, `MainPageBottomAppBar.jsx`, `TravelAreaSelect.jsx` 컴포넌트들이 실제로 디스크 상에서 완전 삭제되어 앱 기동이 불가능했습니다.
* **보정 조치**: 모든 누락 컴포넌트들을 복원했습니다. 특히, 의존성 목록에 존재하지 않아 빌드를 깨트리던 `date-fns` 라이브러리 사용을 전면 파기하고, 프로젝트의 기본 경량화 날짜 패키지인 `dayjs`를 백엔드 달력 데이터 흐름과 완벽히 호환되게 작동하도록 **`AdapterDayjs` 기반의 초경량화 달력**으로 재설계 복원했습니다.

### 2.3. 로그인 페이지(`LoginPage.jsx`) 자바스크립트 문법 및 동시 정의 오류 수정
* **기존 문제**: 
  ```javascript
  const navigate = useNavigate();
  const { login } = useAuth = useAccessToken(); // 문법 에러 (Syntax Error)
  const navigate = useNavigate();              // 변수 중복 정의 에러
  ```
* **보정 조치**: 문법 오류를 올바른 컨텍스트 훅 호출 구문(`const { login } = useAccessToken();`)으로 교정하고 중복된 변수 선언을 깨끗이 소거했습니다.

### 2.4. 여행 계획 생성 시 백엔드 규격 매핑 (400 Bad Request 원천 예방)
* **기존 문제**: 프론트엔드 상에서 저장하는 여행지 객체의 데이터 포맷은 `{ keywords: ["경복궁"] }`과 같이 복수형 배열 형태였으나, 백엔드는 `{ keyword: "경복궁" }` 이라는 단수형 변수가 포함된 객체를 역직렬화 대상으로 기대했습니다. 이를 그대로 송신하면 무조건 `400 Bad Request` 에러와 함께 생성이 불발되는 구조였습니다.
* **보정 조치**: 통신을 전송하기 바로 직전, 임시 배열에서 핵심 키워드를 안심하고 파싱하여 백엔드 DTO 양식에 맞추어 주는 **데이터 매핑 파이프라인(formattedDestinations)**을 구현해 400 Validation Error를 근본적으로 예방했습니다.

### 2.5. 페이지 무단 진입 방지를 위한 세션 라우트 가드 기용
* **기존 문제**: 조원이 복원하는 과정에서 기존에 경로를 안전하게 지켜주던 `PrivateRoute` 컴포넌트를 소거하여 비로그인 외부 사용자가 메인 화면 및 결과 타임라인 화면에 무단 침입해 401 에러를 계속 뿜어낼 수 있는 상태였습니다.
* **보정 조치**: `App.jsx` 의 직관적인 레이아웃 구조를 보존해 주기 위해, 메인 페이지(`MainPage.jsx`)와 결과 페이지(`ResultPage.jsx`) 내부에 **토큰 부재 시 즉시 `/login`으로 안전하게 화면을 추방시키는 세션 가드 스케줄러(`useEffect`)**를 각각 안전하게 장착했습니다.
